# Social Poster

Telegram-бот автопостинга в **Telegram, ВКонтакте, Threads и на сайт (с авто-перепостом в Яндекс Дзен)**.
Ты присылаешь боту идею — он проверяет факты по веб-источникам, пишет адаптированные посты под
каждую площадку, предлагает промпт для картинки и публикует после твоего подтверждения.

## Возможности

- 📝 **Генерация под площадки** — разные тексты под Telegram, ВК и статью для сайта/Дзена
  (не один скопированный, а адаптированные под формат каждой).
- 🔎 **Фактчек с веб-поиском** — перед публикацией бот ищет и **читает источники целиком**
  (`web_search` + `web_fetch`), выносит вердикт. Непроверенное **не публикуется** без явного
  подтверждения.
- 🧑‍⚖️ **Редакторская доводка** — отдельный проход «главреда» вычищает AI-штампы, усиливает хук,
  сверяет факты и формат.
- 🎨 **Промпт для картинки** — бот выдаёт готовый промпт для генератора изображений
  (Nano Banana Pro): живая сцена-метафора, без текста и логотипов.
- 🖼️ **Публикация с фото** — присылаешь фото в чат, бот постит пост с ним.
- ✅ **Гейт подтверждения** — ничего не уходит в эфир без кнопки «Опубликовать».
- 🎛️ **Выбор площадок** — кнопками: 1, 2 или все 3.

## Как работает

```
идея в бот
  → выбор площадок (Telegram / ВК / Threads / Сайт+Дзен)
  → фактчек (Sonnet + web_search + web_fetch) → вердикт
  → черновик (Haiku) → доводка главредом (Sonnet)
  → бот присылает тексты + промпт картинки
  → присылаешь фото → «Опубликовать»
  → Telegram, ВК и Threads — через API; сайт — через /api/news; Дзен забирает из RSS сайта
```

Генерация — через Claude API (модели: Haiku для черновика, Sonnet для фактчека и доводки),
с кэшированием системного промпта для экономии.

## Установка

```bash
git clone https://github.com/makar-nikshonkov/social-poster.git
cd social-poster
pip install -r requirements.txt
cp config.example.json config.json   # затем заполни config.json своими ключами
python3 bot.py
```

## Конфигурация (`config.json`)

| поле | назначение |
|---|---|
| `telegram_bot_token` | токен бота из @BotFather |
| `telegram_chat_id` | id/username канала для публикации (напр. `@my_channel`) |
| `telegram_owner_id` | твой Telegram user id — бот отвечает только тебе |
| `anthropic_api_key` | ключ Claude API (console.anthropic.com) |
| `vk_access_token` | токен сообщества ВК (для текста) / пользователя (для фото) |
| `vk_group_id` | числовой id сообщества ВК |
| `threads_user_id` | id аккаунта Threads (Meta for Developers) |
| `threads_access_token` | long-lived токен Threads, scope `threads_basic` + `threads_content_publish` |
| `site_api_url` | эндпоинт публикации на сайт (см. `api-external.md`) |
| `site_api_key` | ключ доступа к API сайта |

`config.json` хранит секреты и **не коммитится** (в `.gitignore`). Шаблон — `config.example.json`.

## Файлы

- `bot.py` — Telegram-бот: диалог, кнопки, приём фото, публикация.
- `generator.py` — пайплайн генерации (фактчек → черновик → доводка) через Claude API.
- `social_poster.py` — публикаторы: Telegram, ВК, сайт (`/api/news`).
- `api-external.md` — спецификация API сайта для создания новостей.
- `config.example.json` — шаблон конфига.

## Ограничения

- **ВК + фото** требует **пользовательского** токена с правами `photos, wall, groups`
  (токен сообщества фото загружать не может). Текст постится и токеном сообщества.
- **Дзен** не имеет публичного API публикации — статья уходит на сайт, а Дзен забирает её
  из RSS (`/news/rss.xml`). Нужна одноразовая настройка трансляции в Дзен.Студии.
