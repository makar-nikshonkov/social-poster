# Social Poster (Telegram-бот)

Отдельный бот, который работает сам: пользователь шлёт идею в Telegram → бот пишет посты,
проверяет факты, после подтверждения публикует. Видео и Higgsfield тут не участвуют.

## Поток бота

1. Пользователь пишет боту идею/новость.
2. Бот спрашивает площадки (кнопки: ТГ / ВК / Дзен — можно 1, 2 или все 3).
3. Генерация (`generator.py`):
   - **Researcher** (Sonnet + web_search) — проверяет факты, собирает свежие данные.
   - **Writer** (Haiku) — пишет черновики под выбранные площадки.
   - **Critic** (Sonnet) — сверяет с фактами и бренд-войсом, возвращает финал + пометку.
4. Бот присылает готовые тексты + кнопки `[Опубликовать] [Переписать] [Отмена]`.
5. **Гейт:** публикация только по кнопке «Опубликовать». ТГ и ВК — через API, Дзен — в `zen_post.txt`.

## Экономия токенов

- Системный промпт (бренд-войс, правила формата) кэшируется (prompt caching).
- Черновик пишет дешёвый Haiku, дорогой Sonnet — только факт-чек и финальная вычитка.
- RAG и Batch не используются: на текущем масштабе не дают выигрыша (см. историю обсуждения).

## Файлы

- `bot.py` — Telegram-бот (long-poll), состояние диалога, кнопки, публикация.
- `generator.py` — пайплайн генерации через Anthropic API (research → write → critic).
- `social_poster.py` — публикаторы (ТГ/ВК через API, Дзен в файл) + старый CLI dry-run.
- `config.json` — токены: бот ТГ + id канала, `telegram_owner_id`, `anthropic_api_key`, ВК.
- `requirements.txt` — зависимости (`anthropic`, `requests`).

## Запуск

```bash
pip install -r requirements.txt
python bot.py
```

Запускается одинаково локально и на сервере. Для генерации нужен `anthropic_api_key` в config.

## Правила

- Гейт обязателен: публикация только после кнопки подтверждения.
- Не публиковать непроверенные факты: фактчек с веб-поиском обязателен (как с историей WB).
- Если одна площадка упала — остальные публикуются, в конце отчёт.
- `telegram_owner_id` ограничивает бота одним пользователем (иначе отвечает всем).
