## 1. SEO-поля для передачи новости на сайт
**Запрос:** «В передаче новости на сайт проработай поля seo».

API сайта (`impactlaunch.ru/api/news`) поддерживает `meta_title`, `meta_description`, `slug`,
но генератор их не отдавал. Проработали по всей цепочке:

- `generator.py` — у площадки `zen` в схему статьи добавлены обязательные поля:
  - `meta_title` — SEO-заголовок 50–60 знаков, ключ в начале (может отличаться от видимого title);
  - `meta_description` — сниппет 140–160 знаков, законченная фраза с ключами и пользой;
  - `slug` — ЧПУ латиницей, 3–6 слов через дефис.
  - Инструкции добавлены в бренд-войс и в чек-лист критика (пункт №8 — критик доводит SEO).
- `social_poster.py` — `post_site` шлёт SEO-поля в API; добавлена `_clean_slug`
  (чистка ЧПУ до `[a-z0-9-]`, схлопывание дефисов, обрезка до 80 символов).
- `bot.py` — поля пробрасываются в `post_site`; в превью статьи показывается SEO-блок
  (под обязательный гейт — проверка до публикации).
- `external_url` (дедупликация) пока не используется: у бота нет URL первоисточника.

## 3. Перенос на VPS (Docker)
**Запрос:** перенести бота на VPS в Яндекс.Облаке. Решение: Docker-контейнер, `restart: always`.

Подготовлены:
- `Dockerfile` — `python:3.12-slim`, ставит зависимости, запускает `bot.py`; секреты в образ не пекутся.
- `docker-compose.yml` — `restart: always`, `config.json` монтируется томом (только чтение),
  ротация логов (3×10 МБ).
- `.dockerignore` — исключает `config.json`, `.git`, доки, мусор.
- `DEPLOY.md` — рунбук: SSH → Docker → перенос → запуск.
- Код запушен в GitHub: `https://github.com/makar-nikshonkov/social-poster.git`.
  Деплой на ВМ — через `git clone`.

Ключевые решения: секреты вне образа и вне git; long-poll → входящих портов не требуется.

## 6. Диагностика SSH (проблема на стороне сервера)
Попытка входа с мака: TCP до порта 22 проходит (`Connection established`), но сервер **не присылает
SSH-баннер** (`timed out during banner exchange`); `nc` подтвердил — порт принимает соединение, но
данных нет; ICMP-пинг 100% потерь. Вывод: проблема не в ключе, а на стороне сервера/провайдера
(ВМ зависла/не запущена, sshd не отвечает, либо анти-DDoS режет после рукопожатия).
Замечено: IP `31.44.9.168` — не диапазон Яндекс.Облака.

В итоге пользователь подключился по SSH **с Windows-компьютера**.

## 7. Деплой с сервера
Так как SSH работает с Windows, деплой делаем прямо на ВМ:
1. Поставить Docker (`curl -fsSL https://get.docker.com | sudo sh`, добавить в группу `docker`).
2. `git clone https://github.com/makar-nikshonkov/social-poster.git && cd social-poster`.
3. Создать `config.json` на сервере (вставить свои значения, без висячей запятой).
4. `docker compose up -d --build` → `docker compose logs -f` (ждём «Бот запущен»).
5. Остановить локальный `python bot.py`, если где-то ещё крутится (конфликт long-poll).

## Важные примечания
- **Висячая запятая в `config.json`.** В локальном файле после `site_api_key` периодически
  появляется запятая перед `}` — это невалидный JSON, бот упадёт на загрузке. Убрать перед запуском.
- **Перевыпуск секретов.** Токены пересылались через Telegram — рекомендуется перевыпустить
  Telegram-бот-токен (@BotFather → /revoke) и Anthropic-ключ.
- **Один процесс бота.** Telegram отдаёт long-poll апдейт только одному получателю — нельзя держать
  локальный запуск и контейнер на ВМ одновременно.
